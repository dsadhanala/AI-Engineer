# /writer

Tool-agnostic command that edits, drafts, or gives line-by-line feedback on
writing using the canonical `writing-editor` skill. Works in Cursor, Claude,
Codex, Augment, Droids, etc.

## What to do

1. **Read and follow** `~/.agents/skills/writing-editor/SKILL.md`
   as the source of truth for voice (humble, system-thinking, collaborative
   Principal Engineer), the four framing principles, the Principal's Lexicon,
   the workflow, and the genre playbooks. Do not improvise a different voice.

2. **Identify the genre** from the input and apply the matching playbook:
   - Slack message / PR or doc **review-comment reply** → anti-defensive,
     "coordination tax" framing; rewrite first, rationale after.
   - **Career / leadership doc** (promotion packet, level-targeted checkin,
     peer-support letter) → **interview first** to gather metrics, scope, and
     concrete examples; mirror any reference structure provided; lead with
     impact; calibrate to the target level; keep it business-friendly.
   - Design doc, leadership email, status update, essay, letter, etc. →
     follow the general workflow.

3. **Respect the constraints**: never use em dashes; for non-technical readers
   prefer plain language over the lexicon and flag the trade-off;
   sound human, humble and as leader.

4. **Deliver** per the skill: brief editorial overview, categorized feedback
   (or a single rewrite + rationale for short messages), run the Principal
   Check, then offer a final clean draft that is easy to copy.
